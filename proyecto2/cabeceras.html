Cabeceras de HTTP

Cada petición HTTP que el navegador realiza al servidor, se divide en 2 partes:


– Las cabeceras (Headers)


– El cuerpo de la respuesta (Response body)


Cabeceras HTTP más habituales


  1. Server: indica el tipo de servidor HTTP empleado
  2. Age: indica el tiempo que ha estado el objeto servidor almacenado en un proxy cache (en segundos)
  3. Cache-control: lo usa el servidor para decirle al navegador que objetos cachear, durante cuánto tiempo, etc...
  4. Content-Encoding: se indica el tipo de codificación empleado en la respuesta
  5. Expires: indica una fecha y hora a partir del cual la respuesta HTTP se considera obsoleta. Usado para gestionar caché.
  6. P3P: se usa para especificar el tipo de política de privacidad empleado en la web. No está muy extendido.
  7. Location: usado para especificar una nueva ubicación en casos de redirecciones.
  8. Set-Cookie: sirve para crear cookies. Las famosos cookies viajan entre el servidor y el navegador a través de estas cabeceras HTTP.






    Códigos de respuesta

      Son los códigos que se ven cuando el navegador no puede mostrar la página solicitada. El código de respuesta está formado por tres dígitos: el primero indica el estado y los dos siguientes explican la naturaleza exacta del error.


    • CONTENT-TYPE: El Tipo Mime de la respuesta (En caso de que sea una respuesta), o del contenido subido vía POST/PUT (en caso de que sea una Request)
    • CONTENT-LENGTH: El tamaño de la respuesta en octetos (8 bits)
    • ACCEPTED: La solicitud ha sido aceptada, pero el procedimiento que sigue no se ha llevado a cabo
    • PARTIAL INFORMATION: Cuando se recibe este código en respuesta a un comando de GET indica que la respuesta no está completa.
    • NO RESPONSE: El servidor ha recibido la solicitud, pero no hay información de respuesta
    • RESET CONTENT: El servidor le indica al navegador que borre el contenido en los campos de un formulario
